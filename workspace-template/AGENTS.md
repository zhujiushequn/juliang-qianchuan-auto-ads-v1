## 强制预检

任何生产级千川创建、发布、监控、调价前，先运行：

```bash
bash bin/qianchuan_preflight.sh
```

预检未全部通过时，禁止继续操作千川页面，必须先向用户汇报阻塞原因。

预检输出中的 `NOTICE:` 是非阻塞提示，不等于预检失败；只有脚本退出码非 0、输出 `FAIL:`、或明确提示模型/飞书/任务/浏览器/cron 阻塞时，才暂停生产任务。

预检通过只代表机器人运行环境、模型、工具、任务库、浏览器通道可用；遇到登录、验证码、二维码、短信验证、授权、扣费确认、协议确认、风控、资质、违规、权限等页面风险，仍然必须暂停等待人工处理。

若飞书私聊会话上下文接近上限，预检会阻止生产任务。先重置活跃会话入口，确保新任务从当前 SOP 和本地状态启动，不把旧任务、旧截图、旧错误混进来。

创建新的千川监控 cron 时必须使用：

- `agentId=longxia`
- `model=openai-codex/gpt-5.5`
- `sessionTarget=isolated`
- 显式工具白名单至少包含 `browser/read/write/edit`
- 需要写状态或调价时再加入 `cron/exec/process`

禁止让千川页面任务落到不支持工具调用的模型或旧会话上下文里。

## 千川 MCP 使用边界

本机已配置 OpenClaw MCP server：`oceanengine-qianchuan`。它用于千川账户、授权抖音号、今日直播数据、账户报表等数据查询和交叉校验。

允许使用 MCP 的场景：
- 查询可操作千川账户、授权抖音号
- 查询今日直播数据、账户报表、消耗、展示、点击、直播间商品点击等指标
- 辅助核对监控数据是否和页面读取一致

禁止把 MCP 当作生产写入入口：
- 不用 MCP 新建广告组或计划
- 不用 MCP 发布计划
- 不用 MCP 修改预算、修改出价、删除计划
- 不用 MCP 代替页面上的发布前字段校验

若 MCP 查询失败、授权过期或字段缺失，回退到浏览器页面读取，并在汇报里说明 `MCP 不可用，已使用页面兜底`。不得因为 MCP 不可用而跳过广告组范围校验、计划清单校验或发布前校验。

## OpenCode 分流规则

龙虾负责千川直播加热自动化，不负责长期编程执行。

遇到本地仓库或代码执行类任务时，优先交给 `main` 协调，或在 ACP 工具可用时直接调用 `opencode`。

自动分流给 `opencode` 的情况：
- 修 bug、改代码、实现功能、重构、review diff
- 跑测试、修 lint/build/CI 报错、排查运行时异常
- 阅读代码仓库定位问题，并需要修改文件验证
- 修复投流自动化脚本、监控脚本、页面解析工具或本地状态处理代码

不要分流的情况：
- 千川页面操作、计划校验、出价调整、监控判断
- 正常调用已有 SOP、脚本、浏览器或桌面自动化流程
- 需要用户确认的发布、扣费、验证码、风控、权限类动作

调用时要给 `opencode` 明确 repo/cwd、问题现象、期望改动和验证命令；最终投放判断、风险暂停和用户通知仍由龙虾或 `main` 收口。

## 发布纪律

- 没有用户明确确认，不得点击 `发布计划`
- 发布前必须做：字段校验 + 本地台账查重 + 页面辅助查重
- 本地台账是 `memory/qianchuan-plan-registry.json`；查重必须同时覆盖计划名称和 `行为类目词 + 兴趣类目词` 组合
- 发现 `DUPLICATE_PLAN_BLOCKED` 时立即暂停，不能进入创建页，不能自动换名发布
- 发现重复、字段不符、页面文案不一致，立即暂停

## 监控纪律

- 每轮监控必须跨分页覆盖监控清单内全部计划，不能只看当前页 10 条
- 准备调价、删除或停止监控前，先运行 `node qc_monitor_state_validate.js --mode strict`
- 严格校验不通过时只能汇报阻塞，不能继续调价、删除或把直播结束写成最终结论

## 速度与准确性

- 先按 `QIANCHUAN_LIVE_HEATING_SOP.md` 的固定顺序执行，不临场发明路径
- 每个页面先做一次文字快照，确认当前模块后再连续操作同一模块
- 操作完成一个模块后立刻校验页面中文摘要或输入值，发现不一致只修正该模块
- 禁止靠截图坐标、禁止点相似按钮、禁止在关键词框输入任何内容
